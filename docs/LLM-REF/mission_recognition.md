# 任务页面识别系统

## 文件关系

- `autowsgr/ui/mission_page.py` - MissionPage 控制器 + 识别逻辑
- `autowsgr/data/missions.yaml` - 任务数据库 (日常/周常)
- `docs/mission_recognition.md` - 设计规范文档
- `test_pkg/misson_rec/` - OCR 回归测试套件

## 关键坐标 (1280x720 截图实测)

### 按钮扫描

- 扫描列 x=0.91, ROI=(0.86, 0.17, 0.96, 0.95)
- "前往" 蓝色: RGB(30,138,239), 用 Color(15,132,228) + tolerance=40
- "领取" 金色: R>=180 && G>=120 && B<=80, 实际约 RGB(252,220,37)
- "一键领取" 底部 UI 按钮过滤: 检查 x=0.80 处是否也有按钮色 (宽按钮=UI)

### OCR 裁切偏移

**核心发现**: 任务名称文本位于按钮中心 **下方 0.105** 处, 不在同一 y 坐标。

- 名称区域: x=(0.22, 0.60), y=(btn_y+0.105-0.035, btn_y+0.105+0.035)
- 进度区域: x=(0.75, 0.87), y=同上 (但 OCR 效果差, 文字低对比度)

### 任务面板卡片布局

每个任务卡高度约 0.25:
- 卡顶部 (btn_y): 奖励数字 + "前往"/"领取" 按钮
- 卡中部 (btn_y+0.105): **任务名称** (如 "击破敌军主力")
- 卡底部 (btn_y+0.14): 任务目标描述文字

## 模糊匹配策略

`_match_mission_name()` 优先级:
1. 精确匹配
2. 子串匹配 (需长度比>=70%, 取最长候选)
3. Levenshtein 距离 <= 5

子串匹配加了 70% 长度比阈值, 避免短名 "击退敌军舰队" 误匹配含 "10次" 的同名变体。

## 进度 OCR 限制

"任务完成度:XX%" 文字是浅蓝色 (~90,190,242) 绘制在浅灰色 (~236,236,236) 背景上, 对比度极低, EasyOCR 识别率不稳定。当前不严格测试 progress 字段。
